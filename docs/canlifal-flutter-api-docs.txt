1) E-POSTA İLE KAYIT OL
   POST /api/auth/mobile-register
   Auth: Yok (public endpoint)
────────────────────────────────────────────────────────

Doğrudan veritabanına kullanıcı kaydı oluşturur.
Web sitesi auth sistemiyle HİÇBİR ilgisi yoktur.
Hoş geldin kredisi (varsayılan 50) otomatik verilir.
Referral kodu geçerliyse bonus kredi eklenir.
Admin/yönetici kullanıcılar otomatik takip edilir.

Request Body:
{
  "email": "kullanici@email.com",        // ZORUNLU
  "password": "sifre123",                // ZORUNLU
  "name": "Ad Soyad",                    // ZORUNLU
  "username": "kullanici_adi",            // ZORUNLU
  "birthDate": "1995-06-15",             // ZORUNLU (ISO date)
  "birthTime": "14:30",                  // ZORUNLU (HH:mm)
  "referralCode": "ABC12345",            // Opsiyonel
  "preferredLanguage": "tr"              // Opsiyonel (varsayılan: "tr")
}

Başarılı Response (201):
{
  "accessToken": "eyJhbG...",
  "refreshToken": "eyJhbG...",
  "user": {
    "id": "cuid_xxx",
    "email": "kullanici@email.com",
    "name": "Ad Soyad",
    "username": "kullanici_adi",
    "role": "user",
    "credits": 50,
    "jetonBalance": 0,
    "cfcBalance": 0,
    "membership": null,
    "referralCode": "F3A2B1C0"
  }
}

Hata Durumları:
- 400: { "error": "Zorunlu alanlar: email, password, name, username, birthDate, birthTime" }
- 400: { "error": "Geçersiz e-posta adresi" }
- 400: { "error": "Bu e-posta adresi zaten kayıtlı" }
- 400: { "error": "Bu kullanıcı adı zaten alınmış" }
- 429: { "error": "Çok fazla istek. Lütfen biraz bekleyin." }
- 500: { "error": "Kayıt başarısız" }

3) GOOGLE İLE GİRİŞ / KAYIT
   POST /api/auth/mobile-google
   Auth: Yok (public endpoint)
────────────────────────────────────────────────────────

Flutter tarafında google_sign_in paketinden alınan idToken gönderilir.
Kullanıcı yoksa otomatik SQL kaydı oluşturulur.
Kullanıcı varsa mevcut hesaba Google bağlanır ve giriş yapılır.
Web sitesi Google SSO ile HİÇBİR İLGİSİ YOKTUR.

Flutter Kodu:
  final GoogleSignIn googleSignIn = GoogleSignIn();
  final account = await googleSignIn.signIn();
  final auth = await account!.authentication;
  final idToken = auth.idToken; // ← Bunu API'ye gönder

{

========================================================
  TOKEN KULLANIMI
========================================================

accessToken  → 7 gün geçerli
refreshToken → 30 gün geçerli

Tüm korumalı endpoint'lerde header'a ekle:
  Authorization: Bearer <accessToken>

accessToken süresi dolduğunda:
  POST /api/auth/mobile-refresh ile yenile.
  401 alırsan kullanıcıyı login ekranına yönlendir.

Önerilen Dio Interceptor akışı:
  1. Her istekte Authorization header ekle
  2. 401 alınca refreshToken ile yenile
  3. Yeni token'larla isteği tekrarla
  4. Refresh de 401 dönerse → login ekranı


========================================================
  ONESIGNAL ENTEGRASYONU
========================================================

Login/register sonrası:
  OneSignal.login(user.id);

Logout'ta:
  OneSignal.logout();

OneSignal App ID: 578518ed-7b16-46a9-a1e6-7692d3ba55d8

Push notification data payload yapısı:
{
  "type": "dm" | "cfc" | "live" | "fortune",
  "targetPath": "/chat/userId" | "/live" | ...,
  "targetId": "related_id",
  "urgent": true | false
}

========================================================
  ÖNEMLİ NOTLAR
========================================================

1. Tüm endpoint'ler doğrudan SQL veritabanına kayıt
   oluşturur — web sitesi auth sistemiyle ilgisi YOKTUR.

2. Google ile giriş yapan kullanıcıya otomatik:
   - Kullanıcı adı oluşturulur (email'den)
   - Profil fotoğrafı Google'dan çekilir
   - Hoş geldin kredisi verilir
   - Admin/yönetici otomatik takip edilir

3. TikTok ile giriş yapan kullanıcının emaili:
   tiktok_{open_id}@canlifal.app formatındadır.

4. isNewUser alanı (Google/TikTok): Kullanıcı yeni mi
   yoksa mevcut hesaba giriş mi yaptığını gösterir.
   true ise → onboarding/profil tamamlama gösterilebilir.

5. cfcBalance tüm auth response'larında döner.

6. Rate limit: IP bazlı, çok fazla istek → 429 hatası.

7. Tüm tarih alanları ISO 8601 formatındadır.

8. null değerler JSON'da null olarak döner.
